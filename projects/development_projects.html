SI 507 FINAL PROJECT

Quick Facts

GitHub Repo: Link
Front End: Bootstrap
Back End: Python with Flask
Timeline: Mar. - Apr. 2019

Motivation

This project was the culmination of many skills we learned in the intermediate python programming course I took. It allows users to build a "party" of Pokemon for the traditional Pokemon RPG games, and tells them stats about the party as they add members. Flask and SQLAlchemy are used to handle reading and writing to the SQLite database. The class was not focused on web or graphic design so the frontend ended up very simple, and relies mostly on Bootstrap's default styling. All details about how to clone and run the code locally are included in the repo's README.

SEMESTER PLANNING APP

Quick Facts

GitHub Repo: Link to come
Front End: React
Back End: Python with Django, hosted on Heroku
Timeline: Jun. 2019 - Present

Motivation

This Semester Planning project grew out of my own need as well as what I learned from other students when I was working as a research assistant at the University of Michigan School of Information's Learning, Education, and Design (LED) Lab. The LED lab had created a "Learning Analytics" dashboard and piloted it with three UM classes, and I interviewed some of those students at the end of the semester about their experiences.

We saw that the dashboard was able to help students plan and monitor their progress for one class, but expressed a strong desire for a tool to help them plan their work, both short- and long-term, across all their classes. I spun those findings into my own project here.

The goal of this web app is to give students a tool to visualize their work load day-to-day and week-to-week across an entire semester for all of their classes. Ideally it will help students avoid unexpectantly overloading themselves and give them a greater sense of agency over their time and studies.
